## 주요 기능  

### 🔹 사용자 의도파악

사용자가의 질의가 어떤 대화를 목적으로 하는 것인지 파악

- 사용자의 질의를 통해 [일반, DB, 로컬파일, 웹] 검색이 필요한지 파악
- 질의에 필요한 함수 호출
➡️ 사용자가 함수 호출 명시 없이 자연스러운 대화를 통해 필요한 함수 호출

---

### 🔹 자연어 기반 DB 조회

사용자가 Git Hub 프로젝트 관련으로 내용 질문 시

- 해당 질의를 Encoder를 통해 Embedding Vector로 변환 
- 사용자의 Embedding Vector와 DB에 있는 Embedding Vector를 비교하여 유사도가 높은 Top k를 뽑음 
- Top k 문서중 유사도가 가중 높은 Context와 사용자 질의를 LLM모델의 입력으로 사용

➡️ 사용자가 모든 정보를 조회할 필요 없이 자연어를 통해서 해당 데이터를 찾을 수 있음


----

### 🔹 코드 리팩토링
에이전트는 해당 내용을 바탕으로 사용자가 원하는 코드로 리팩토링

- 현재 어떤 구조로 구성되어 있는지  
- 어떤 기능과 모듈이 포함되어 있는지  
- 개선이 필요해 보이는 부분은 무엇인지  

스스로를 소개하듯 응답하는 기능  

➡️ 협업 관점에서 온보딩 속도 향상, 지식 베이스를 유지할 수 있도록 함
